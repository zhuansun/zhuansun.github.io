+号在java中作用

发表于2022-10-31|更新于2022-10-31|JAVA数据库MYSQL
|阅读量:

+号在java中作用

  • 作为运算符，当作用于数值型的时候，是运算符

  • 作为连接符，当作用于字符串的时候，是连接符

+号在mysql中的作用

不能作为连接符

select first_name+last_name from user order by id desc;


期望的结果：
-------------
颛孙鹏程
张三
李四



实际的结果
-------------
0
0
0

为什么会这样呢？ 看下面就知道了。

只能作为运算符

两个操作数都为数值型

当两个操作数都为数值型的时候，做加法运算。

select 100+20;

结果
-----------
120

一个操作数为数值型，另一个是字符型

当一个操作数为数值型，另一个是字符型的时候，会试图将字符型转换成数值型：

  • 如果转换成功，然后在做加法运算。

  • 如果转换失败，会默认转换成0，然后在做加法运算。

select 'zhuansun'+20;
结果
-----------
20



select '120'+20;
结果
-----------
140

两个操作数都是字符型

当两个操作数都是字符型的时候，会将字符型默认转换成0，然后在做加法运算。这也就解释了我们之前说的：为什么不能作为连接符。

select 'zhuansun'+'wenqing';

结果
-----------
0

一个操作数为null，另一个操作数为字符型或者数值型。

当两个操作数任意一方为null的时候，不管另一个是什么，结果都是NULL。

select null+20;

结果
-----------
NULL

NULL值与任何值连接，结果都是NULL

select null+100;

结果：
--------
NULL


# 假设score可能为null
select CONCAT(first_name,',',last_name,',',socre) from user order by id desc;

结果
----------
颛孙,鹏程,98
张,三,90
null
李,四,87
null
null

那么在第二个查询语句结果中出现null的原因是因为score是null，然后与 first_name 和 last_name 进行连接，不管这两个值有没有值，结果都是null。
